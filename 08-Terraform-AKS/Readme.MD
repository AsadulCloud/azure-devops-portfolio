# 08 ‚Äî AKS + ACR + CI/CD Infrastructure via Terraform

Full cloud-native CI/CD infrastructure deployed using Terraform. No manual clicks. Everything as code.

---

## What This Project Does

Deploys a complete Azure Kubernetes Service (AKS) infrastructure including:

- **AKS Cluster** ‚Äî Kubernetes cluster in Spain Central
- **Azure Container Registry (ACR)** ‚Äî Stores Docker images
- **Agent VM** ‚Äî Self-hosted Azure DevOps agent for CI/CD
- **Networking** ‚Äî VNet, subnets, NSG, public IPs
- **RBAC** ‚Äî AKS pulling images from ACR via Managed Identity

---

```
Developer pushes code
        ‚Üì

        ‚Üì
```

## Step by Step ‚Äî Deploy This Infrastructure

### Prerequisites
- Azure CLI installed and logged in (`az login`)
- Terraform >= 1.9.0
- Active Azure subscription
- Azure DevOps organization

---

### Step 1 ‚Äî Login to Azure

```bash
az login
```

---

### Step 2 ‚Äî Create Service Principal

```bash
az ad sp create-for-rbac \
  --name "terraform-sp" \
  --role="Contributor" \
  --scopes="/subscriptions/YOUR_SUBSCRIPTION_ID"
```

Save the output ‚Äî you need `appId`, `password`, and `tenant`.

---

### Step 3 ‚Äî Export Credentials

```bash
export ARM_CLIENT_ID="your appId"
export ARM_CLIENT_SECRET="your password"
export ARM_TENANT_ID="your tenant"
export ARM_SUBSCRIPTION_ID="your subscription id"
```

To make permanent:
```bash
echo 'export ARM_CLIENT_ID="your appId"' >> ~/.bashrc
echo 'export ARM_CLIENT_SECRET="your password"' >> ~/.bashrc
echo 'export ARM_TENANT_ID="your tenant"' >> ~/.bashrc
echo 'export ARM_SUBSCRIPTION_ID="your subscription id"' >> ~/.bashrc
source ~/.bashrc
```

---

### Step 4 ‚Äî Create Backend Storage

```bash
bash setup-backend.sh
```

Copy the printed storage account name into `backend.tf`.

---

### Step 5 ‚Äî Create `terraform.tfvars`

```hcl
prefix                       = "votingapp"
location                     = "Spain Central"
vnet_address_space           = ["10.0.0.0/16"]
subnet_address_space         = ["10.0.1.0/24"]
aks_subnet_address_space     = ["10.0.2.0/24"]
admin_username               = "azureuser"
admin_password               = "YourSecurePassword!"
vm_size                      = "Standard_B1s"
acr_sku                      = "Basic"
aks_node_count               = 1
aks_node_vm_size             = "Standard_B2s_v2"
os_disk_caching              = "ReadWrite"
os_disk_storage_account_type = "Standard_LRS"

tags = {
  Environment = "Staging"
  Owner       = "Asadul"
  Project     = "AKS-CICD"
}
```

---

### Step 6 ‚Äî Initialize and Deploy

```bash
terraform init
terraform plan
terraform apply
```

This takes **8-12 minutes** ‚Äî AKS cluster creation takes the longest.

---

### Step 7 ‚Äî Connect kubectl to AKS

```bash
az aks get-credentials \
  --resource-group votingapp-rg \
  --name votingapp-aks

kubectl get nodes
```

---

### Step 8 ‚Äî Install ArgoCD

```bash
kubectl create namespace argocd
kubectl apply -n argocd \
  -f https://raw.githubusercontent.com/argoproj/argo-cd/stable/manifests/install.yaml

# Expose ArgoCD
kubectl patch svc argocd-server \
  -n argocd \
  -p '{"spec": {"type": "LoadBalancer"}}'

# Get ArgoCD password
kubectl -n argocd get secret argocd-initial-admin-secret \
  -o jsonpath="{.data.password}" | base64 -d
```

---

### Step 9 ‚Äî Set Up Azure DevOps Agent on VM

SSH into the VM:
```bash
ssh azureuser@VM_PUBLIC_IP
```

Install agent:
```bash
mkdir myagent && cd myagent
wget https://vstsagentpackage.azureedge.net/agent/3.x.x/vsts-agent-linux-x64-3.x.x.tar.gz
tar zxvf vsts-agent-linux-x64-3.x.x.tar.gz
./config.sh
./run.sh
```

---

### Step 10 ‚Äî Assign AcrPull Role to AKS

```bash
KUBELET_ID=$(az aks show \
  --name votingapp-aks \
  --resource-group votingapp-rg \
  --query identityProfile.kubeletidentity.objectId -o tsv)

ACR_ID=$(az acr show \
  --name votingappacr0001 \
  --query id -o tsv)

az role assignment create \
  --assignee $KUBELET_ID \
  --role AcrPull \
  --scope $ACR_ID
```

---

### Step 11 ‚Äî Clean Up

```bash
terraform destroy
```

## Errors I Hit and Fixed

| Error | Cause | Fix |
|---|---|---|
| `Standard_DS2_v2 not allowed` | VM size not available in Spain Central | Changed to `Standard_B2s_v2` |
| `ServiceCidrOverlapExistingSubnetsCidr` | Service CIDR overlapped with VNet | Added `service_cidr = "10.2.0.0/16"` |
| `PublicIPCountLimitReached` | Free tier limit of 3 public IPs | Removed ArgoCD public IP temporarily |
| `ImagePullBackOff` | AcrPull role on wrong identity | Assigned role to kubelet identity |
| CRLF line endings in shell script | Windows line endings in `.sh` file | Added `.gitattributes` with `*.sh eol=lf` |
| ACR name taken | `votingappacr` already exists globally | Added `0001` suffix |

## Production Improvements

For a production deployment the following would be added:

- **NGINX Ingress Controller** ‚Äî single LoadBalancer for all services (cost efficient)
- **cert-manager** ‚Äî automatic TLS/HTTPS certificates
- **Private AKS nodes** ‚Äî no public IPs on nodes (more secure)
- **Azure Key Vault** ‚Äî secrets management instead of environment variables
- **Horizontal Pod Autoscaler** ‚Äî auto-scale pods based on load
- **Multiple node pools** ‚Äî separate pools for different workloads

---

## Key Learnings

- AKS nodes should always be private in production ‚Äî use NGINX Ingress for external access
- AcrPull role must be assigned to `kubelet_identity` not `cluster identity`
- Service CIDR must not overlap with VNet address space
- Remote backend state must use a different `key` for each Terraform project
- `terraform.tfvars` must never be committed to Git
